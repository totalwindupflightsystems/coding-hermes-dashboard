🏗️ Architecture

🌞 Helios — Constellation Model

  • Multi-agent architecture: Coordination Layer + Domain Agents (Network, DB, App) + Subagents
  • Agents communicate via NATS JetStream (shared incident log)
  • 5-tier RBAC (Viewer → Super Admin), MFA, HashiCorp Vault
  • PostgreSQL + MySQL + SQLite, Redis, config hot-reload
  • ~18% shipped

⚔️ Susanno — Deterministic Core

  • Simple pipeline: Alerts → Dedup → Incident → Runbook Engine → Notify
  • Deterministic-first: everything works with ai.enabled=false
  • AI copilot is optional, provider-agnostic, never executes — only investigates + proposes
  • Event-sourced incident timeline is the source of truth
  • 100% shipped

🔑 Key Design Differences

🧠 Philosophy

  • Helios: "Multi-agent constellation" — domain-specific AI agents that observe, orient, decide, act. The AI is the architecture.
  • Susanno: "Deterministic-first" — core loop works without AI. AI is an optional copilot that investigates and proposes. Never executes.

🐛 Dedup Quality

  • Helios: Check-then-insert pattern → TOCTOU race condition
  • Susanno: Atomic UPSERT on unique index → Race-free
  • Explicitly called out in Susanno's PLAN.md §2 as a Helios bug fix

📦 Package Hygiene

  • Helios: Duplicate packages — logger and logging, agent and agents
  • Susanno: GOAL.md §7: "Before creating a package, grep for an existing one — Helios drowned in duplicates"
  • 43 small, focused packages — one concept, one package

🔌 MCP (Agent Interop)

  • Helios: No MCP whatsoever
  • Susanno: Full MCP server (stdio + streamable HTTP) + MCP client (copilot queries external servers)
  • 3 read tools + 2 write tools, same store layer as REST — zero drift

🔄 Gaps: What each should borrow from the other

Susanno → Helios (10 features to port)

  • ✅ AI copilot (provider-agnostic, guardrailed)
  • ✅ MCP server + client
  • ✅ Atomic dedup (fix the TOCTOU race)
  • ✅ 6 more notification providers
  • ✅ Slack-native coordination
  • ✅ Native on-call schedules/rotations
  • ✅ Public status page
  • ✅ SLO burn-rate alerting
  • ✅ Postmortem generation
  • ✅ HA with leader election

Helios → Susanno (3 features to consider)

  • 🔶 Enterprise auth — 5-tier RBAC, MFA, Vault (when real users demand it)
  • 🔶 Config hot-reload — fsnotify-based (low effort, high DX value)
  • 🔶 Multi-agent constellation — if single-copilot model proves insufficient at scale

🏁 Bottom Line

Susanno IS the Helios you wanted. Same product, rebuilt with the lessons from Helios's over-engineering. The 15:1 code ratio isn't because features were dropped — it's because the architecture is simpler and every line is wired.

Helios's value now: reference library (specs, test patterns, enterprise auth). Susanno's value: the shipping product.
